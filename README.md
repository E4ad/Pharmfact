# Mission Management UI

Standalone React/TypeScript localStorage prototype for the mission management workflow.

The existing Python project in `/Users/admin/Documents/facturation` is used only as a functional reference. Its HTML templates and invoice templates are not modified by this project.

Invoices are rendered in full React through `src/features/invoices/InvoiceTemplate.tsx` and the print route `/invoices/:invoiceId/print`. PDF downloads are generated by the Express/Playwright backend through `POST /api/invoices/:id/pdf`, using the current localStorage snapshot sent by the browser. `GET /api/invoices/:id/pdf` is also available for backend/QA seed invoices such as `inv_seed_1`.

The API healthcheck is available at `GET /api/health` and returns `{ "status": "ok" }`. Vite proxies `/api` to `http://127.0.0.1:${API_PORT:-3001}`. The React prototype backend is the Express API on port `3001`; port `5000` may be occupied by another local macOS service and is not used by this prototype.

Address suggestions are served by `GET /api/geocode?q=...`. By default, the backend proxies Nominatim/OpenStreetMap for Canadian addresses. The provider can be adjusted with `GEOCODE_ENDPOINT`, `GEOCODE_COUNTRYCODES`, `GEOCODE_LANGUAGE`, `GEOCODE_USER_AGENT`, or disabled with `GEOCODE_DISABLED=true`.

Run after installing a JavaScript package manager:

```bash
cd /Users/admin/Documents/mission-app
npm install
npm run server
npm run dev
```

Direct connectivity checks:

```bash
curl http://127.0.0.1:3001/api/health
curl http://127.0.0.1:5173/api/health
```

## PDF QA checklist

1. Copy `.env.example` if local port overrides are needed.
2. Start the backend: `npm run server`.
3. Start the frontend: `npm run dev`.
4. Verify proxy health through the frontend: `npm run qa:proxy`.
5. Verify direct backend PDF: `curl -i http://localhost:3001/api/invoices/inv_seed_1/pdf` returns `Content-Type: application/pdf` and a body starting with `%PDF`.
6. In the app, open `Factures` and click `Télécharger PDF`.
7. With the backend stopped, the UI should display: `Le PDF n’a pas pu être généré. Vérifiez que le serveur est démarré puis réessayez.`

Automated checks:

```bash
npm test
npm run qa:proxy
npm run test:e2e
```

## Captures UX automatiques

Installation:

```bash
npm install
npx playwright install chromium
```

Avant de lancer les captures, démarrer le backend API si nécessaire:

```bash
npm run server
```

Lancer l’application:

```bash
npm run dev
```

Puis générer les captures:

```bash
npm run screenshots
```

Ou lancer automatiquement Vite puis les captures:

```bash
npm run screenshots:auto
```

Capturer seulement les missions:

```bash
npm run screenshots -- --page missions
```

Capturer seulement mobile:

```bash
npm run screenshots -- --viewport mobile
```

Capturer tous les viewports:

```bash
npm run screenshots -- --all-viewports
```

Options utiles:

```bash
npm run screenshots -- --base-url http://127.0.0.1:5173
npm run screenshots -- --output screenshots
npm run screenshots -- --timestamp
npm run screenshots -- --headed
npm run screenshots -- --debug
npm run screenshots -- --dark
npm run screenshots -- --light
npm run screenshots -- --no-full-page
npm run screenshots -- --keep-animations
npm run screenshots -- --active-pharmacien ph_amelie
```

IDs configurables pour les routes dynamiques:

```bash
SCREENSHOT_MISSION_ID=mis_completee SCREENSHOT_INVOICE_ID=inv_seed_1 npm run screenshots
```

Les fichiers PNG sont générés dans:

```text
/screenshots
```

Le script capture les pages principales en desktop et mobile:

```text
welcome
activity
missions-new
missions-dashboard
missions-edit
missions-drawer-open
invoices
invoice-print
financial
pharmacies
pharmaciens
exports
```

Le script continue si une page ou un état interactif échoue et affiche un résumé final avec les réussites et erreurs.

Si le backend API est nécessaire, démarrez-le avant les captures:

```bash
npm run server
```

Avec le script automatique, l’option suivante affiche un avertissement si `/api/health` n’est pas joignable:

```bash
npm run screenshots:auto -- --with-backend
```

Pour capturer le drawer mission ouvert, les composants React contiennent des identifiants stables:

```tsx
data-testid="mission-row"
data-testid="mission-drawer"
```
